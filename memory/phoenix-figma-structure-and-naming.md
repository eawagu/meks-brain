## Summary

Mandatory Figma file organization and naming conventions linking design work directly to product management artefacts (Change Specs, Stories, Epics). Naming is deterministic: same change spec always produces same Figma path, enabling automated validation.

## Key Points

- Hierarchy: Workspace (platform cluster) → Team (PLATFORM-{code}) → Project (CAPABILITY-{code}) → File (EPIC-{code}-EP{digits}) → Page (STORY) → Frame (AC path)
- Journey files use separate pattern: JOURNEY-{platform code} with FLOW-{journey code} pages
- Taxonomy validation enforced at handoff: 100% compliance required
- Legacy files must be migrated as work touches them
- Two file types: Epic files (contain stories) vs. Journey files (cross-epic flows)
