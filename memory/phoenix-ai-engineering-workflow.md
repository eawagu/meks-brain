## Summary

Defines the AI-powered engineering workflow for Project Phoenix. Core philosophy: Claude as force multiplier, not replacement; engineers remain fully accountable. Covers quality standards, code review processes, deployment guidelines, security protocols, and productivity best practices.

## Key Points

- Claude assists with coding, review, testing; engineer owns all output
- Quality standards and code review processes for AI-assisted development
- Deployment guidelines and security protocols
- Accountability frameworks and responsibility assignment
- Integration with Change Specs and design handoff
- Productivity best practices while maintaining code quality
