## Summary

Initial February 27, 2026 executive overview: [[Transaction Switching Platform]] as Layer 1 payment kernel with market-agnostic Spine + Module architecture. 16 transaction types, 6-phase migration strategy with shadow mode (14-day 0% discrepancy gate).

## Key Points

- Spine + Module architecture
- 16 transaction types (ASYNC/SYNC)
- 6-phase migration strategy with shadow mode
- Shadow mode gate: 14-day 0% discrepancy
- Six shared dependencies: CBA, Cosmos, Loom, Treasury/FX, Notifications, Kafka
- Multi-tenant separation
- PII vaults
- Market-agnostic expansion model
- Does NOT address direct card scheme connectivity (deferred to Card Issuance/Acceptance)
