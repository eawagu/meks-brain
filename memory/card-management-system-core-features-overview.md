## Summary

Structured overview of CMS core feature domains. Two deployment models: Processor Model (multi-tenant) and Bank/Issuer Model (direct). Covers issuer management, card program config, card production lifecycle, PIN management, card controls, cardholder experience (mobile app), transaction monitoring, and regulatory compliance. Integration points: Authorization Engine, EMV Data Prep, TSP, Notifications.

## Key Points

- Two deployment models: Processor (multi-tenant) and Bank/Issuer (direct)
- Card types: debit, prepaid, virtual
- Card controls: spending limits, MCC restrictions, geographic/channel restrictions
- PIN management: set/change/reset, secure SMS delivery, optional biometric
- Portal UI: issuer mgmt, card program setup, production tracking, controls config, monitoring dashboards
- Cardholder mobile: activation, PIN change, block/unblock, limits config, history, dispute initiation
- Regulatory: KYC/KYB verification, AML screening (via Loom/TSP), data protection, audit trails
