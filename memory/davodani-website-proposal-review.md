## Summary

An internal critique of [[Dosh Services]]' proposal to redesign the [[DavoDani Microfinance Bank]] website. The review finds the proposal adequate as a sales document but too thin to approve, identifying seven critical gaps across technical specification, compliance, scope management, post-launch support, IP ownership, timeline, and vendor references.

## Key Points

- No tech stack specified — CMS choice (WordPress vs custom), hosting ownership, form backend integration, and domain/email setup all undefined
- Compliance language is vague — needs explicit CBN guidelines and NDPA compliance, data storage/protection specifics, and pre-launch pen test
- Scope is open-ended — no defined revision rounds, no change request process, no acceptance criteria for triggering final 20% payment
- Post-launch support (30 days) undefined — unclear what's covered (bug fixes vs content updates) and no retainer option after day 30
- IP ownership not mentioned — need written confirmation that full code and design ownership transfers to DavoDani on final payment
- 8-week timeline has no milestone dates tied to deliverables, and no clause for delivery slippage
- LAPO Microfinance reference is unverifiable — no metrics, no contact name provided
