Vendor Evaluation & RFP (Request for Proposal) is the structured process by which an organization selects a technology vendor for a significant procurement. A properly-run process includes:

- Documented business and technical requirements
- Defined evaluation criteria (functional fit, regulatory compliance, integration capability, TCO, vendor stability, support model)
- Shortlisting methodology (how candidates make the long list, how the short list is derived)
- Formal RFP with consistent question set across vendors
- Scoring matrix with weighted criteria
- Reference checks and proof-of-concept where warranted
- Contract negotiation including SLAs, exit clauses, and data ownership

## Failure Mode: Category Errors From Undefined Criteria
When evaluation criteria are not well-defined, candidate lists drift into category errors — products that do not belong in the same evaluation get listed alongside true competitors. Per [[DDMFB_2026_IT_Strategy_Review]], [[DavoDani Microfinance Bank]]'s [[Core Banking System]] candidate list included [[Microsoft Dynamics NAV]] — an ERP, not a CBS. This is diagnostic of a missing or weak shortlisting methodology.

## DDMFB 2026 — Application of Concept
The DDMFB review recommended commissioning a proper needs assessment and RFP process for the ₦100M CBS decision, with a published vendor evaluation matrix and TCO analysis covering implementation, training, and post-go-live support.
