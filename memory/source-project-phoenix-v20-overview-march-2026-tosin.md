## Summary

[[Project Phoenix]] is the transformation of [[Moniepoint]] from locally built products into country-agnostic platform capabilities, supported by Execution Governance, top-tier UX standards, extreme reliability, and AI Accelerated Engineering. Business teams become fully country- and segment-owned. Principle: "Platforms build durable capabilities. Countries build durable businesses. Both are accountable. Neither is confused."

## Key Points

### 1. Platform Capabilities — Six Clusters
- **Business Banking Platforms:** Card Acceptance & Processing, Payment Gateway (Monnify+), POS Hardware & Device Platforms, Card Issuance & Processing, Cross-Border Payments.
- **Digital Banking Platforms:** Spend Management Platform, Account Payments, Money Management Platform, Value Added Services (VAS), Rewards Platform.
- **Transaction Switching & Processing ([[TSP]]):** foundational infrastructure — all card, account, VAS processing routes through this cluster. Sub-platforms: Card Switching & Processing, Account Switching & Processing, VAS Switching & Processing, Integrations.
- **Credit Platforms:** Credit Platform.
- **Customer Platforms:** Mobile App, Frontend Web, POS App, CRM & Product Instrumentation, Design System & Experience Enablement (treated as first-class platform domain).
- **Banking Operations Platforms:** Banking & Ledger Platform ([[CBA]]), Sales Management Tools (MonieCRM), Customer Support Platform (MonieDesk), Finance Systems, Field Verification Platform, MOOS, Financial Crime Prevention ([[Loom]]), Business Management Platform (Moniebook), Identity/Onboarding & Shell.
- Platform Accountability: capability architecture, API contracts, service uptime/latency, error budgets, customer experience performance.
- Design System & UI Frameworks: atomic design system (Phoenix), shared UI component libraries, Web/Mobile/POS frameworks.

### 2. Execution Governance
- Outcomes cascade across group → entity → country → platform → team with defined review cadences: Annual (strategy), Quarterly (commitments), Monthly (execution/priorities).
- **Epic documentation required:** problem definition, customer segment, success metrics & expected impact, dependencies & milestones, post-launch measurement. "If it is not in the artifact, it does not exist."

### 3. Top-Tier User Experience
- Operationalized through atomic design system "Phoenix" — cohesive, consistent interfaces across every surface.
- Satisfaction metrics: NPS, CSAT. Behavioural metrics: time on task, completion rate, drop-off rates across critical journeys.

### 4. Extreme Reliability
- **99.99% minimum uptime** across all core platforms — hard accountability, not aspirational.
- Defined P99 latency targets per platform; explicit error budgets with ownership; expanded automated test coverage; transparent metrics; increased engineering talent density.
- **Zero tolerance for unowned incidents.**

### 5. AI Accelerated Engineering (Mandate, Not Recommendation)
- All engineering development must be AI-first.
- Workflows: code gen, test gen, documentation, code review augmentation, refactoring, debugging & incident analysis.
- [[Eywa]] — institutional intelligence: AI skills encode workflows across every function; context is institutional, not personal; model-agnostic architecture, LLM-portable by design; **skills are the durable asset, not the underlying model.**
- AI is default operating layer for every function (engineering, product, design, operations, finance, people, legal, compliance).

### 6. Business Teams — Entity & Country Ownership
- Three current entities consuming platforms: [[Moniepoint MFB]] Nigeria (Business/Retail/Digital/Operations), [[TeamApt]] Ltd Nigeria, Kenya (first international entity).
- Group-level functions (e.g., Market Research) serve all entities, inform platform decisions, belong to no single country/entity GM.
- **Business owns** revenue/profitability, market share, unit economics, local partnerships, regulatory relationships, sales/distribution, operations, compliance.
- **Platforms own** capability architecture, API contracts, UX systems, service reliability, error budgets, customer experience performance, country-agnostic reusability.
- Entity selects behaviors from platform's capability superset via runtime configuration — new entity = configuration, never rebuilding. This is "the compounding return of platformization."

### 7. Risks & Mitigation
| Risk | Description | Mitigation |
|---|---|---|
| Platform becomes choke point | Centralised ownership = dependency bottleneck slowing country teams | AI-first velocity; systems-thinking PMs; mandatory capability documentation; clear API contracts |
| Premature abstraction | Capabilities generalised before patterns understood → brittle/over-engineered layers | Build for known use cases first; promote to platform only when pattern stable across multiple entities |
| Exception creep | Country-specific exceptions accumulate in core platform logic, eroding country-agnosticism | Strict separation of logic extensions vs data extensions (Uber DOMA model); governance gate on all exceptions |
| Leadership fatigue | Senior leaders disengage or revert to pre-Phoenix patterns | Regular leadership conference checkpoints; MOOS accountability; explicit OKR linkage to Phoenix milestones |

### 8. Talent Density — Calibration Moment
- Phoenix is deliberate calibration moment. Complexity surfaces quality of judgment.
- **Two Primary Dimensions Tested:**
  - **Customer Obsession** — every platform capability anchored to real customer problem. Technically coherent work disconnected from customer reality will not meet bar.
  - **Craft** — first-principles mastery required for country-agnostic systems. Pattern-matching and tribal knowledge insufficient.
- **Three Threshold Dimensions (non-negotiable):** No Ego, Candor, Integrity. No performance level compensates for failure on any.
- **Observable Dimensions:** Systemisation, Urgency & Ownership, Candor (repeated — also threshold).
- "Phoenix will raise the bar on what good looks like. It will also make visible people who have been operating below that bar without it being apparent. Both are expected, both will be addressed. We intend to use it."

### Success Outcomes
- Platform: faster country launches, capability reuse, shorter cycle time, improved deployment frequency, lower incident rates, platforms not becoming bottlenecks.
- Customer & Business: higher user satisfaction, consistent UX across countries, profitable country operations, increased talent density, compounding institutional knowledge.
