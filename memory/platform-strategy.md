## Overview

Platform Strategy is Moniepoint's strategic framework underpinning [[Project Phoenix]]. Core thesis: **"One Platform, Many Markets"** — a unified platform architecture serves multiple country markets via configuration rather than code duplication.

## Strategic Vision

- **One Platform, Many Markets** — unified architecture as common backbone
- **Capability-based composition** — market products built by configuring/composing platform capabilities
- **No code duplication across markets** — configuration-driven adaptation
- **Capabilities as unit of reuse** — not services, not code
- **Configuration > code forks** — market differentiation happens in config, not branches

## Technical Architecture Rules

- **Microservices** — small, independently deployable services
- **API-first** — all capabilities exposed via stable contracts
- **Event-driven** — async messaging as primary integration pattern
- **Config-driven** — market-specific behavior in configuration

## Global Expansion Alignment

Platform Strategy directly enables [[International Expansion]]:
- Nigeria (base)
- UK (diaspora, business banking)
- Kenya (mobile money, via [[Sumac Microfinance Bank]] acquisition)
- Future markets: South Africa, Morocco, DRC (per [[Class Strategy]])

## Governance

- **ARB (Architecture Review Board)** — governs technical architecture decisions
- **Compliance framework** — cross-market compliance engine
- **KPIs** — measured platform adoption, market activation time, capability reuse rate
