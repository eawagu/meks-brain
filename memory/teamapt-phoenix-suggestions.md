## Summary
Process suggestions and actionable guidelines for operationalizing Project Phoenix, focusing on PM workflow plugin, mandatory outputs per work type, and cross-platform project coordination.

## Key Points
- PM plugin as Phoenix's operational workflow engine with five starter-kit categories: New Product, New Feature, New Project, Migration, Update
- Mandatory outputs per work type: Change Spec (always required), C4 Architectural Diagram Inputs, Participant Matrix
- "Project" category for cross-platform coordination (work spanning multiple products with external parties)
- Development workflow enforcement: small phases, roadmap tracking, phase archival
- Data standardization as critical prerequisite: standardized schema for customers, capabilities, entry points, flows, user journeys
- Risk mitigation: parallel information-gathering streams (product managers + engineering leads) to de-risk schema definition
- Immediate actions: define schema, schedule PM sessions, sync with engineering, establish working sessions
