## Summary

Comprehensive framework for writing system specifications to Phoenix standards. Covers index, process, template, formatting, examples, and alignment checklist across 6 files.

## Key Points

- 11-section template: Executive Summary, System Overview, Business Context, Capabilities, Regulatory, Architecture, Integration, Performance, Operational Model, Roadmap, Stakeholders
- Dual output: markdown (.md) for Obsidian + Word (.docx) for executive distribution
- File naming: [SystemName]_System_Specification.md (PascalCase with underscores)
- Phoenix alignment checklist: platform thinking, API-first, configuration over customization, automation, observability, security
- Gold-standard references: Aptent, Directory Service, TACHA specs
- AI-assisted drafting: provide context files, reference template, fill gaps interactively
